## Overview

The Terminal API provides functions for creating and managing terminal instances, drawing frames, and controlling cursor behavior.

## Types

### Terminal

```typescript
interface Terminal {
  readonly backend: Backend;
  buffers: readonly [Buffer, Buffer];
  current: number;
  readonly viewport: Viewport;
  readonly viewportArea: Rect;
  hiddenCursor: boolean;
  frameCount: number;
}
```

### Frame

```typescript
interface Frame {
  readonly buffer: Buffer;
  readonly area: Rect;
  readonly count: number;
  cursorPosition?: Position;
}
```

### CompletedFrame

```typescript
interface CompletedFrame {
  readonly buffer: Buffer;
  readonly area: Rect;
  readonly count: number;
}
```

### Viewport

```typescript
type Viewport = 'fullscreen' | 'inline' | 'fixed';
```

### TerminalOptions

```typescript
interface TerminalOptions {
  readonly viewport: Viewport;
}
```

## Terminal Functions

### createTerminal

Creates a new terminal instance with the specified backend.

```typescript
const createTerminal = (
  backend: Backend,
  options?: Partial<TerminalOptions>
): Terminal
```

<ParamField path="backend" type="Backend" required>
  The backend implementation for rendering
</ParamField>

<ParamField path="options" type="Partial<TerminalOptions>">
  Optional terminal configuration
  - `viewport`: Viewport mode ('fullscreen' | 'inline' | 'fixed')
</ParamField>

**Returns:** A new `Terminal` instance

---

### terminalDraw

Draws a frame by executing the render function and updating the display.

```typescript
const terminalDraw = (
  terminal: Terminal,
  renderFn: (frame: Frame) => void
): CompletedFrame
```

<ParamField path="terminal" type="Terminal" required>
  The terminal instance to draw to
</ParamField>

<ParamField path="renderFn" type="(frame: Frame) => void" required>
  Function that renders content to the frame's buffer
</ParamField>

**Returns:** A `CompletedFrame` with the rendered buffer and metadata
